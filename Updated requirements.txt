Requirement Document
Title: Marketing Text-to-SQL Agent with RBAC Simulation (POC)
1. Objective

Develop a Proof of Concept (POC) conversational analytics application that:

Converts natural language questions into SQL queries
Executes queries on a local SQLite marketing dataset
Returns both tabular results and visual insights
Demonstrates Role-Based Access Control (RBAC) behavior
Provides a persona simulation capability to showcase how different users see different data
Is deployable using Streamlit Community Cloud for easy demo sharing

The solution should be lightweight, secure, and extensible, enabling future migration to enterprise data platforms such as Snowflake, Databricks, or Azure SQL.

2. RBAC Personas for Demo

The application must simulate multiple personas with controlled data access.

Demo Persona	Role Purpose	Access Scope
Marketing Analyst	Analyst view	Can see campaign and channel performance metrics. Restricted from sensitive finance rollups beyond approved scope
Marketing Manager	Manager view	Can view all marketing metrics across campaigns and channels
Finance Viewer	Finance view	Can view spend and revenue summaries but restricted from detailed campaign-level operational metrics
Executive	Leadership view	Can view aggregated performance summary only
Admin	Full access	Can view all data columns and all rows
3. Simulation Mode Requirement (Critical for Demo)

To effectively demonstrate RBAC behavior without implementing enterprise authentication, the application must include a Simulation Mode control.

3.1 Simulation Control Placement

A persona selector must be visible in the top-right corner of the application UI.
